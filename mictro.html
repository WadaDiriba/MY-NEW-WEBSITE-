Micro-frontends with Flutter

Building modular applications using micro-frontend architecture with independent deployment and team autonomy

Architecture Overview

Module Federation

Independent Flutter modules that can be developed, tested, and deployed separately while working together as a single application.

Dynamic Imports Shared Dependencies Runtime Integration

Shell Application

Main container application that orchestrates micro-frontends, handles routing, and provides shared services.

Router Integration State Management Dependency Injection

Shared Infrastructure

Common services, state management, and UI components shared across all micro-frontend modules.

Implementation Guide

1

Project Structure Setup

Create a monorepo structure with independent Flutter packages for each micro-frontend.

# Project Structure
libs/
  ├── shell/          # Main application
  ├── auth/           # Authentication module
  ├── dashboard/    # Dashboard module
  ├── profile/        # User profile module
  └── shared/        # Shared dependencies
2

Module Configuration

Configure each module with independent pubspec.yaml and dependency management.

# auth/pubspec.yaml
name: auth_module
dependencies:
  flutter:
    sdk: flutter
  shared:
    path: ../shared
  riverpod: ^2.0.0
3

Shell Application Router

Implement dynamic routing in the shell application to load micro-frontends on demand.

class MicroFrontendRouter {
  static Map<String, WidgetBuilder> get routes => {
    '/auth': (context) => AuthMicroFrontend(),
    '/dashboard': (context) => DashboardMicroFrontend(),
    '/profile': (context) => ProfileMicroFrontend(),
  };
}
4

Shared State Management

Implement Riverpod providers in the shared package for cross-module state management.

// shared/providers/auth_provider.dart
final authProvider = StateNotifierProvider<AuthNotifier, AuthState>((ref) {
  return AuthNotifier();
});

class AuthNotifier extends StateNotifier<AuthState> {
  AuthNotifier() : super(AuthInitial());
  
  void login(String email, String password) {
    // Authentication logic
  }
}

Key Benefits

Team Autonomy

Independent teams can develop, test, and deploy their micro-frontends without coordination overhead.

Independent Deployment

Deploy features independently without full application redeployment, enabling faster release cycles.

Technology Flexibility

Different teams can use different Flutter versions or experimental features in their modules.

Fault Isolation

Issues in one micro-frontend don't affect the entire application, improving overall stability.
